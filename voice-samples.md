# Voice Samples

This file teaches the loop how Jamshed actually writes — not just abstract rules, but pattern-matchable examples. The Sunday sweep reads this on every run and matches voice when drafting.

**Until you fill in real samples, the loop falls back to the voice rules in `playbook.md`.** Drafts will be technically correct but generically toned. Adding 2–3 real samples is the single highest-leverage upgrade you can make.

## How to add a sample

1. Pick a post you've written that you think is *most* you — not your best-performing, your most authentic
2. Paste the full text below in a new `## Sample N` block
3. Annotate what makes it yours in the "**What this sample teaches the loop**" subsection
4. Repeat for 2–3 samples covering different post types (build log, teardown, news take)

## How the loop uses this file

The Sunday sweep loads this file. For each draft, it:

- Pattern-matches sentence rhythm and hook style against the samples
- Hard-rejects any draft containing a "phrases I'd never say" item
- Biases word choice toward the "phrases that sound like me" allowlist
- Avoids structural moves seen in the anti-samples

**Maintenance:** add a new sample anytime you write a post you're proud of. Replace the oldest sample once you have 5+. The loop only needs 2–3 active to work.
